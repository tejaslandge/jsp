pom.xml Configuration

Key Features

  • Packaging type: WAR (Web Application Archive)
  • Embedded Tomcat 7 for easy deployment
  • Port configured to 8081 (avoids conflicts)

<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0

    <modelVersion>4.0.0</modelVersion>

    <groupId>com.example</groupId>
    <artifactId>demo</artifactId>
    <version>1.0.0</version>
    <packaging>war</packaging>

    <dependencies>
        <dependency>
            <groupId>com.mysql</groupId>
            <artifactId>mysql-connector-j</artifactId>
            <version>8.0.33</version>

    </dependencies>

        <finalName>demo</finalName>

STEP 7

Run JSP Application

This command starts the embedded Tomcat server on port 8081. Your application will be accessible through your web browser.

Start Tomcat Server 
mvn tomcat7:run

Access in Browser

Application URL 
http://localhost:8081/jsp-demo/index.jsp?number=5

Important Notes

What NOT to Do

  • Don't use Live Server (JSP not supported)
  • Don't run WAR files directly
  • Avoid port 8080 (common conflicts)

Key Points

  • JSP cannot run directly in browser
  • JSP needs Tomcat server to execute
  • WAR files are deployed, not executed

Database Connection (db.jsp)

This file establishes a connection to MySQL database. Include it in all JSP pages that need database access.

db.jsp 
<%@ page import="java.sql.*" %>
<%
String url = "jdbc:mysql://localhost:3306/your_db?useSSL=false&serverTimezone=UTC";
String user = "root";
String password = "password";

Class.forName("com.mysql.cj.jdbc.Driver");
Connection con = DriverManager.getConnection(url, user, password);
%>
P0 - START HERE

READ - Fetch Records

Learn to display data from database first

Why start with READ? Understanding how to fetch and display data is fundamental. Once you can see data, you'll better understand how to add, delete, and update it.

ADD - Insert Record

Now learn to add new data to the database

addUser.jsp HTML Form with Validation

Form with Client-side Validation 
<!DOCTYPE html>
<html>
<head>
    <title>Add User</title>
    <script>
        function validateForm() {
            let name = document.forms["userForm"]["name"].value;
            let email = document.forms["userForm"]["email"].value;

            if (name.trim() === "" || email.trim() === "") {
                alert("All fields are required");
                return false;
            }
            return true;
        }
    </script>
</head>
<body>
    <h2>Add User</h2>
    <form name="userForm" action="insertUser.jsp" method="post" 
          onsubmit="return validateForm()">
        Name: <input type="text" name="name"><br>
        Email: <input type="email" name="email"><br>
        <input type="submit" value="Save">
    </form>
    <a href="listUsers.jsp">View Users</a>
</body>
</html>

insertUser.jsp Server-side Processing

Insert Data with Validation 
<%@ page import="java.sql.*" %>
<%@ include file="db.jsp" %>

<%
String name = request.getParameter("name");
String email = request.getParameter("email");

// Server-side validation
if (name == null || email == null || 
    name.trim().equals("") || email.trim().equals("")) {
    out.println("Invalid Input");
    return;
}

PreparedStatement ps = con.prepareStatement(
    "INSERT INTO users(name, email) VALUES (?, ?)"
);
ps.setString(1, name);
ps.setString(2, email);

int rows = ps.executeUpdate();
ps.close();
con.close();

if (rows > 0) {
    response.sendRedirect("listUsers.jsp");
} else {
    out.println("Insert failed");
}
%>
P2 - THIRD

DELETE - Remove Record

Learn to remove data - simpler than updating

Why before UPDATE? Delete is simpler as it only needs the ID. Once you understand delete, update will be easier to grasp.

deleteUser.jsp - Delete Record 
<%@ page import="java.sql.*" %>
<%@ include file="db.jsp" %>

<%
String idParam = request.getParameter("id");

if (idParam == null) {
    out.println("Invalid User ID");
    return;
}

int id = Integer.parseInt(idParam);

PreparedStatement ps = con.prepareStatement("DELETE FROM users WHERE id=?");
ps.setInt(1, id);
ps.executeUpdate();

ps.close();
con.close();

response.sendRedirect("listUsers.jsp");
%>
P3 - ADVANCED

UPDATE - Modify Record

Most complex operation - combines READ and INSERT concepts

Why last? Update requires fetching data (READ), displaying it in a form (like ADD), and then modifying it. Master the other operations first!

editUser.jsp Load Data for Editing

Fetch and Display User Data 
<%@ page import="java.sql.*" %>
<%@ include file="db.jsp" %>

<%
String idParam = request.getParameter("id");
if (idParam == null) {
    out.println("Invalid User ID");
    return;
}

int id = Integer.parseInt(idParam);
PreparedStatement ps = con.prepareStatement("SELECT * FROM users WHERE id=?");
ps.setInt(1, id);
ResultSet rs = ps.executeQuery();

if (!rs.next()) {
    out.println("User not found");
    return;
}
%>

<html>
<body>
    <h2>Edit User</h2>
    <form action="updateUser.jsp" method="post">
        <input type="hidden" name="id" value="<%= rs.getInt("id") %>">
        Name: <input type="text" name="name" value="<%= rs.getString("name") %>" required><br>
        Email: <input type="email" name="email" value="<%= rs.getString("email") %>" required><br>
        <input type="submit" value="Update">
    </form>
    <a href="listUsers.jsp">Back to List</a>
</body>
</html>

<%
rs.close();
ps.close();
con.close();
%>

updateUser.jsp Process Update

Update Database Record 
<%@ page import="java.sql.*" %>
<%@ include file="db.jsp" %>

<%
String idParam = request.getParameter("id");
String name = request.getParameter("name");
String email = request.getParameter("email");

// Server-side validation
if (idParam == null || name == null || email == null ||
    name.trim().equals("") || email.trim().equals("")) {
    out.println("Invalid input data");
    return;
}

int id = Integer.parseInt(idParam);

PreparedStatement ps = con.prepareStatement(
    "UPDATE users SET name=?, email=? WHERE id=?"
);
ps.setString(1, name);
ps.setString(2, email);
ps.setInt(3, id);

int rows = ps.executeUpdate();
ps.close();
con.close();

if (rows > 0) {
    response.sendRedirect("listUsers.jsp");
} else {
    out.println("Update failed");
}
%>

CRUD Workflow (Learning Priority)

P0 - READ
listUsers.jsp (displays all records) ‚Üê Start Here!
P1 - ADD
addUser.jsp ‚Üí insertUser.jsp ‚Üí listUsers.jsp
P2 - DELETE
listUsers.jsp ‚Üí deleteUser.jsp ‚Üí listUsers.jsp
P3 - UPDATE
listUsers.jsp ‚Üí editUser.jsp ‚Üí updateUser.jsp ‚Üí listUsers.jsp

Learning Path Explanation

  • P0 (READ): Learn to see data first - it's fundamental to understanding databases
  • P1 (ADD): Once you can view data, learn to insert new records
  • P2 (DELETE): Simpler than update - just needs an ID to remove a record
  • P3 (UPDATE): Most complex - combines READ (fetch data) + ADD (modify data)

Best Practices & Security

Always Follow

  • Use PreparedStatement - Prevents SQL Injection
  • Server-side Validation - Never trust client input
  • Close Resources - Prevent memory leaks
  • Use Hidden Fields - For IDs in update forms

Important for Exams

  • Know the complete file structure
  • Understand the workflow between files
  • Remember to include db.jsp
  • Always redirect after update/delete

Note: This JSP-only approach is for learning purposes. Real-world applications use Servlets, DAO pattern, and MVC architecture for better code organization.
